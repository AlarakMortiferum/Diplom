# Результаты тестирования мобильного приложения «Мобильный хоспис»

## Краткий итог
В рамках дипломного проекта было выполнено:
- ручное тестирование приложения;
- подготовка тестовой документации;
- автоматизация ключевых пользовательских сценариев;
- подготовка отчета о результатах автоматизированного тестирования.

## Что было автоматизировано
Автоматизированы следующие проверки:
- успешная авторизация;
- негативная авторизация с пустыми полями;
- открытие экрана About;
- открытие раздела Our Mission и раскрытие первой цитаты;
- открытие раздела News и раскрытие первой новости;
- создание новости;
- редактирование новости.

## Что показала автоматизация
Автотесты подтвердили корректную работу основного функционала приложения:
- пользователь может успешно авторизоваться;
- приложение корректно обрабатывает часть негативных сценариев авторизации;
- доступны экраны About и Our Mission;
- работает базовый блок работы с новостями: открытие, создание, редактирование.

## Найденные проблемы
В ходе проверки были выявлены дефекты приложения. Один из подтвержденных дефектов:
- при вводе неверных логина и пароля отображается сообщение `Something went wrong. Try again later.` вместо ожидаемого сообщения о неверных учетных данных.

Дефекты оформлены в GitHub Issues.

## Сравнение ручного и автоматизированного тестирования
### Ручная проверка
Преимущества:
- удобно исследовать интерфейс;
- можно быстро замечать визуальные и UX-проблемы;
- подходит для первичного обследования приложения.

Недостатки:
- занимает больше времени;
- зависит от внимательности тестировщика;
- сложнее воспроизводить одинаковые шаги много раз.

### Автоматизированная проверка
Преимущества:
- быстрое повторное выполнение тестов;
- воспроизводимость сценариев;
- удобство регрессионной проверки;
- возможность получать отчеты Allure.

Недостатки:
- требует предварительной настройки окружения;
- требует поддержки тестового проекта;
- нестабильные или меняющиеся элементы UI усложняют автоматизацию.

## Временные затраты
Оценка по итогам работы:
- ручная проверка по чек-листу занимает заметно больше времени, так как каждый сценарий выполняется вручную;
- автоматизированный прогон ключевых сценариев занимает считанные минуты после подготовки окружения.

Итог: автоматизация особенно эффективна для повторных прогонов, регрессии и подтверждения исправлений.

## Вывод
Автоматизация второй части дипломного проекта выполнена. Реализован набор UI-автотестов для ключевых функций приложения, подготовлены Allure-результаты и оформлены найденные дефекты.
